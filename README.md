# VectorMind MCP

VectorMind 是一个给 AI 编程助手用的“项目记忆” MCP。

它的作用很简单：让 AI 记住这个项目现在到底要怎么做、哪些需求已经改过、哪些旧说法已经过时，避免后面继续开发时又把功能改回老版本。

比如：

> 一开始任务申请需要上级审核，后来改成申请直接通过。  
> 之后再改审核相关功能时，AI 不应该又按旧需求改回“需要上级审核”。

VectorMind 就是为了解决这类问题。

---

## 它能帮你做什么

- 记住当前正在做的需求。
- 记住每次代码修改的原因。
- 记住最新的业务决定。
- 把已经过时的旧需求标记掉。
- 新会话开始时，让 AI 先恢复项目上下文再继续做。
- 减少“AI 忘了前面改过什么”“AI 又按旧需求写代码”的情况。

所有数据默认保存在项目本地：

```text
<your-project>/.vectormind/
```

---

## 安装

最简单的方式是直接用 npx：

```bash
npx -y @coreyuan/vector-mind
```

也可以全局安装：

```bash
npm install -g @coreyuan/vector-mind
```

当前版本：

```text
1.0.41
```

---

## Codex 配置示例

在 `~/.codex/config.toml` 里添加：

```toml
[mcp_servers.vector-mind]
type = "stdio"
command = "npx"
args = ["-y", "@coreyuan/vector-mind"]
```

配置后重启 Codex。

## 推荐用法

### 1. 新会话开始

让 AI 先读取项目记忆：

```text
先用 VectorMind 恢复这个项目的上下文，再继续做。
```

### 2. 开始新需求前

让 AI 先记录当前需求：

```text
先记录这个需求：任务申请提交后直接通过，不需要上级审核。
```

### 3. 改完代码后

让 AI 记录这次为什么改：

```text
把这次改动原因同步到 VectorMind。
```

### 4. 需求发生变化时

一定要明确告诉 AI 这是“最新决定”：

```text
这是最新决定：任务申请不再需要上级审核，申请后直接通过。请写入 VectorMind，并标记旧审核需求已过时。
```

这是最关键的一步。  
只要最新决定被记录下来，后续新会话恢复上下文时，AI 就会优先看到当前决定，而不是旧需求。

---

## 常用能力

你不需要手动记所有工具名，正常告诉 AI 要做什么即可。

如果需要明确指定，可以用这些说法：

- “恢复项目上下文”
- “记录当前需求”
- “同步这次改动意图”
- “保存这个最新决定”
- “把旧需求标记为过时”
- “查一下之前关于这个功能的记录”
- “找一下这个函数/模块在哪”

---

## 多项目使用建议

如果你同时打开多个项目，建议在对话里明确项目路径，例如：

```text
这个任务的项目路径是 H:\2025\YourProject，请 VectorMind 使用这个 project_root。
```

这样可以避免不同项目的记忆混在一起。

---

## 更新后不生效怎么办

如果你刚发布或安装了新版本，但 MCP 行为没有变化：

1. 重启 Codex / VS Code / Claude 等客户端。
2. 开一个新会话。
3. 确认 MCP 配置仍然指向：

```bash
npx -y @coreyuan/vector-mind
```

---

## 一句话总结

VectorMind MCP 不是普通笔记工具。  
它是给 AI 编程助手用的项目记忆，让 AI 在长期开发中优先相信“最新决定”，而不是把旧需求又改回来。
